## Part A：資料蒐集與整理（70%）

### 資料來源

- **平臺**：政府資料開放平臺 <https://data.gov.tw/>
- **資料集**：YouBike 2.0 臺北市公共自行車即時資訊
- **下載網址**：`https://tcgbusfs.blob.core.windows.net/dotapp/youbike/v2/youbike_immediate.json`
- **格式**：JSON（每筆包含站名、行政區、經緯度、總停車格、可借車輛、可還空位、更新時間）

### 統一任務規格

| 步驟 | 要求 |
|:----:|:-----|
| 1 | 資料取得（rvest 爬蟲或 API） |
| 2 | 資料清理（dplyr、tidyr、stringr） |
| 3 | 資料儲存（CSV 輸出） |
| 4 | 探索分析（至少 4 張 ggplot2 圖表） |
| 5 | Quarto 報告（完整流程說明，至少 6+ 頁 PDF） |

### 分析任務

1. **站點分布**：各行政區（`sarea`）站點數量分布，哪些區域站點最密集？
2. **使用率分析**：計算各站「可借車輛 / 總停車格」比例，辨識高需求與低需求站點
3. **供需失衡**：找出「幾乎全滿」（可借 < 3）與「幾乎全空」（可還 < 3）的站點，分析其空間分布特性
4. **資料品質**：檢查資料更新時間（`mday`），是否有站點資料異常停止更新？`act` 欄位為 0 的站點有哪些？

### 圖表要求（至少 4 張）

| # | 圖表類型 | 說明 |
|---|---------|------|
| 1 | 長條圖 `geom_col` | 各行政區站點數量排序 |
| 2 | 直方圖 `geom_histogram` | 全站可借車輛比例分布 |
| 3 | 散佈圖 `geom_point` | 總停車格 vs 可借車輛（以行政區著色） |
| 4 | 長條圖 `geom_col` + `coord_flip` | 可借車輛最少之 Top-15 站點 |

## Part B：臺鐵訂票程式實做（30%）

### 任務說明

參考 `orig/thsr.R`（高鐵訂票程式）與 `orig/thsr.pdf`（原理介紹），
撰寫**臺鐵網路訂票**自動化程式，並在 RStudio 中實際展示。

臺鐵訂票網址：<https://www.railway.gov.tw/tra-tip-web/tip/tip001/tip123/query>
臺鐵退票網址：<https://tip.railway.gov.tw/tra-tip-web/tip/tip001/tip115/query>

### 必要功能

1. **chromedriver 自動安裝**：偵測 Chrome 版本、下載對應 chromedriver
2. **WebDriver session 管理**：啟動與關閉瀏覽器（`start_driver` / `stop_driver`）
3. **DOM 操作工具**：`wait_for`、`fill`、`click` 等輔助函式
4. **訂票流程**：自動填寫身分證、車次、車站、日期 → 處理驗證碼 → 查詢 → 選擇車次 → 確認
5. **退票流程**：輸入訂位代碼 → 查詢 → 取消訂位 → 確認
6. **截圖存檔**：關鍵步驟截圖（至少 3 張）
7. **錯誤處理**：`tryCatch` 包裝，逾時與重試機制

### 展示要求

- 在 RStudio 中執行，**實際完成一次訂票與退票**
